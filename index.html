Privacy Policy

Effective Date: June 11, 2026

Thank you for using AI Detection and Counting of Cotton Whitefly, developed by ICAR-Central Institute for Cotton Research (ICAR-CICR), India.

This Privacy Policy explains how the application handles user data, permissions, storage, and security. We are committed to protecting user privacy and ensuring transparency regarding the operation of the application.

The application is designed for agricultural and research-related purposes, specifically for detecting and counting insects from images captured or uploaded by users.

1. Application Overview

AI Detection and Counting of Cotton Whitefly is an offline Android application that uses Artificial Intelligence (AI) and Machine Learning (ML) technologies to:

  • Detect insects from images
  • Count detected insects
  • Generate results locally on the device
  • Save detection reports in CSV format
  • Allow users to manually share generated CSV files

The application performs all processing locally on the user's device without requiring internet connectivity.

2. Information We Collect

The application is designed with privacy-first principles.

We do NOT collect, store, transmit, or share any personally identifiable information (PII) or sensitive user information.

Specifically, the application:

  • Does not require account registration
  • Does not require user login
  • Does not collect names, phone numbers, or email addresses
  • Does not access contacts or messages
  • Does not collect location information
  • Does not use cookies
  • Does not use analytics services
  • Does not use advertising SDKs
  • Does not track user behavior
  • Does not upload images or reports to any server
  • Does not transmit user data over the internet

3. Permissions Used by the Application

3.1 Camera Permission

The camera permission is used to capture images of insects or crops for AI-based detection and counting.

The captured images remain stored locally on the user's device unless manually deleted or shared by the user.

The application does not upload or transmit camera images to any external server.

3.2 Storage / Media Permission

Storage or media access permission is used to:

  • Select existing images from the gallery
  • Save generated CSV reports
  • Store processed result images locally
  • Allow users to manually export or share files

The application only accesses files selected by the user or files generated by the application itself.

No storage data is collected remotely.

4. Offline Processing and Data Handling

The application operates completely offline.

All AI detection, counting, image processing, and CSV generation activities are executed locally on the Android device using embedded machine learning models.

No user content, including:

  • Images
  • Detection results
  • CSV files
  • Device information

is uploaded, synchronized, or shared with external servers.

Internet access is not required for the primary functionality of the application.

5. Data Storage

Any generated files, including:

  • Captured images
  • Uploaded images
  • Detection result images
  • CSV reports

are stored locally on the user’s device storage.

Users maintain full control over these files and may:

  • Delete files
  • Move files
  • Share files manually
  • Backup files independently

The application does not maintain cloud backups or remote storage systems.

6. Data Sharing

The application does not automatically share any user data with third parties.

Users may voluntarily share generated CSV reports or images using Android sharing features such as:

  • Email
  • Messaging applications
  • Cloud storage apps
  • Bluetooth or file transfer

Such sharing actions are initiated solely by the user.

ICAR-CICR is not responsible for data shared externally by users through third-party applications or services.

7. Third-Party Services

This application does not integrate with third-party analytics, advertising, tracking, or marketing services.

Specifically, the application does not use:

  • Google Analytics
  • Firebase Analytics
  • Advertising SDKs
  • User profiling systems
  • Behavioral tracking systems

The application is intended to function independently without collecting user data.

8. Security

Since all operations are performed offline and data remains on the local device, the risk of remote data exposure is minimized.

However, users are responsible for maintaining the physical and digital security of their devices, including:

  • Device passwords
  • Screen locks
  • Secure file sharing practices

While reasonable efforts are made to ensure application reliability and security, no digital system can be guaranteed to be completely secure.

9. Children’s Privacy

The application does not knowingly collect personal information from children under the age of 13 or any other users.

As no personal data collection occurs, the application presents minimal privacy risk to children.
